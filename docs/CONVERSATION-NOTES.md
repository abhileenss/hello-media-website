## Project in one paragraph

Hello Media is a UAE-headquartered creative marketing agency with a global network. This repo is the v2 rebuild of their marketing site, replacing a v1 that Chaadi (brand partner) felt didn't capture the brand. The site is static HTML + CSS + JS, generated by a Python script (`scripts/build_pages.py`), deployed on Vercel, with 43 pages and a motion language built around the brand's five symbols (Pause / H / Smile / Pixel / Wink). Live at https://hello-media-website.vercel.app (deploy-protected).

## Key decisions locked in

### Brand
- **Colours**: `#000000` / `#FFFFFF` / `#E6062F` (brand red — to confirm Pantone with Chaadi after full brand-guideline sign-off)
- **Fonts**: Archivo Black (display), Barlow Condensed (caps/eyebrow), Inter (body) — these are **closest-free substitutes** for Geometr415 Blk + EngraversGothic BT. Graphics dept to produce side-by-side comparison + paid-font price for Management decision
- **Logo**: currently a stylised approximation; Graphics dept to export clean SVG (logomark separately from wordmark) from the AI source files
- **Motion language**: Pause (hard stop) / Smile (arc up) / Wink (quick twist) / Pixel (blink). Already implemented via custom CSS easings
- **Hero layout**: image-dominant, full-bleed, text panel bottom-left (fixed Chaadi's "half-image" feedback). Chaadi also wants an **edge-to-edge** variant without any frame — to build as A/B

### Copy
- Pulled from the brand PDF (`Brand Assets/Web/Text.pdf`)
- Original homepage title had a typo "WELCOM TO OUR HOME" — shipped corrected as "Welcome to our home"
- Hero subline ("Marketing: shockingly complex — let the experts handle the volts") is under review; three alternatives proposed, Management to pick
- Stub copy on legal/CSR/FAQ/etc. was deliberately **differentiated per page** so the footer doesn't read as mass copy-paste
- Whimsy dialled back on legal/CSR — kept full voice on hub + service pages
- Content philosophy: **show the idea, not just the final result** — case studies use the full Problem / Analysis / Solution / Result format, with photos and videos where possible (Chaadi's explicit ask)

### Architecture
- **Static site** — no framework. Python generator, plain HTML output
- Every CSS/JS asset is cache-busted with `?v=<timestamp>` auto-stamped at build time
- Vercel caching is currently `no-cache, no-store, must-revalidate` during active development (re-enable caching after launch)
- **CMS decision still open** — Chaadi keeps mentioning "will do this via the CMS." Currently no CMS. Needs Tech + Management decision before Phase 2
- **PR workflow** (Chaadi asked what this means): explained in plain terms — proposing a change, getting it reviewed before going live, vs pushing directly. Recommend using PRs once more than one dev is working on the repo

### Ops
- **Vercel**: currently under a personal account — transfer to a Hello Media org pending (P1, Management)
- **Vercel Deploy Protection**: currently ON — means the live site is 403 to anyone not signed in. Blocks my (Claude's) end-to-end verification. Tech to disable
- **Domain**: Chaadi's recommendation is **`hellomedia.site`** (`.com` + `.net` taken; `.ae` alone reads too local). Management to confirm
- **Cookie banner**: confirmed **NOT needed** (Chaadi). Just a footer link + clause in T&C
- **Analytics / form backend / DNS**: Tech to choose and execute
- **GitHub repo**: ownership is currently `abhileenss/hello-media-website` — Management to decide if this moves under a Hello Media org

### Responsibility model
- "Client" as an owner label was wrong (Hello Media isn't the client — they're the company). Replaced in tracker with role labels:
  - **Management** — strategic decisions, content sourcing, approvals
  - **Graphics** — logo exports, font comparison, image cropping + idea-preserving reworks (with nano-banana where needed), mock awards, brochure
  - **Chaadi** — brand direction, final sign-off on design + copy, page-by-page review meeting
  - **Web Dev** — implementation (me, or whoever picks up next)
  - **Tech / Ops** — Vercel ownership, DNS, form backend, analytics, monitoring
  - **Marketing** — social account creation across all platforms under one consistent name
  - **Legal** — privacy, T&C, cookie statement, copyright
  - **Each team member** — own LinkedIn URL, own photo, own bio

### Accessibility + SEO + Security
- Real alt text on every image
- Skip-to-content link + keyboard focus-visible outlines
- `prefers-reduced-motion` respected — all animations killed if user has it on
- Security headers via `vercel.json`: CSP, HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy (camera/mic/geo/FLoC off)
- OpenGraph + Twitter meta per page
- Homepage title carries positioning: _"Hello Media — Creative marketing agency, UAE + global"_

### Arabic version
- Not built. Deferred. Go/no-go from Management before public launch.

- `100vh → 100svh` on hero for iOS
- `vercel.json` set to `no-cache` on CSS / JS / HTML during dev
- Auto cache-bust `?v=<timestamp>` on every build

### Session 5 — April 22 2026 (unblocked engineering sweep)

Worked through the ten items in `NEXT-SESSION-PROMPT.md` — everything that didn't need partner input. Pure scaffolding, no content decisions.

- **Hero A/B**: `?hero=edge` variant live on `/`. Nav goes transparent, `.hero__media` fills 100svh, top-edge darkening ramp keeps nav legible. Documented in `docs/HERO-VARIANTS.md`. Default hero unchanged.
- **Slogan readability**: bumped eyebrow weight 500→700, size 0.7rem→0.95rem, tightened letter-spacing, added text-shadow + drop-shadow on the red square. Separate mobile rule at ≤600px.
- **Social icons**: 9-icon inline-SVG row (FB / IG / Snap / X / TikTok / YT / Pinterest / Flickr / LinkedIn) in footer above `.footer__bottom`. 34×34 px wrapper, 20 px icon, red hover, proper `aria-label` per icon. Handles are `#` until Marketing provides URLs.
- **Cookie + terms**: footer "Cookies" renamed "Cookie statement". Terms stub now carries the "by using this site you agree … have no claim" clause per Chaadi. No popup added — confirmed unwanted.
- **Case study template**: new `.case-hero` + `.case-section` + `.case-card` components; `/what-we-did/sample/` seeded as placeholder; hub now lists cards via `CASE_STUDIES` list (auto-switches to grid when >1 entry).
- **sitemap.xml + robots.txt**: sitemap auto-generated at build (50 URLs) with placeholder canonical `https://hellomedia.site/`. `robots.txt` allows all, blocks `/motion-preview/`, references sitemap.
- **Web manifest + PNG icons**: 180/192/512 PNGs generated from `favicon.svg` via cairosvg, stored in `/assets/icons/`. `manifest.webmanifest` + `apple-touch-icon` now wired into every page shell.
- **Country pages**: 6 pages scaffolded at `/where-we-are/<uae|uk|usa|brazil|china|egypt>/`. Each: country hero image (`contact-<country>.jpg`), "Indicative location" address, placeholder phone, back link to the world map. Map pins now link to these pages via a "Visit →" link in the tooltip (tooltip gains pointer-events and a grace-timer hide so the link is clickable).
- **Motion preview**: new `/motion-preview/` page (not in nav, `noindex,nofollow`, excluded from sitemap + robots). Showcases Pause / Smile / Wink / Pixel / Reveal primitives and an "Replay intro" button that clears sessionStorage.
- **Image focal points**: `FOCAL_POINTS` dict in `scripts/build_pages.py`, applied inline as `object-position: X Y` on every hero, block, service-card and page-header image. Defaults to `center` when a slug is missing.

Local crawl: 44 pages, 92 assets, 0 broken. Served with `python3 -m http.server 8765`.
